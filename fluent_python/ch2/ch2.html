Built-in Sequencess

Sequence types built in C:

  • Container sequences
    • Can hold items of different types
    • Includes list, tuple, and collection.queue
    • Holds references to the objects it contains
  • Flat Sequences
    • Hold items of one simple type:
    • str, bytes, array.array
    • Stores contents in its own memory space

Flat sequences are more compact, but they are limited to holding primitive machine values.

Every python object in memory has a header with metadata. For a float, there are two metadata fields and a value field:

  • ob_refcnt: The objects reference count
  • This is the number of objects that are linked to this memory address
  • For instance, if we say a=1 then b=a, there will be two references to the object 1
    • It is actually not that simple, but that is a clean way of thinking about it
  • We can use sys.getrefcount(b) to help retrieve this, but again the number is not what we expect
  • ob_type: A pointer to the object’s type
  • ob_fval: A C double holding the value of the float

On 64-bit python, each of the above field takes 8 bytes. That is why an array (not a list, but array.array) of floats is more compact than a tuple of floats. For an array, there is only one object holding the raw values for the floats. For a tuple, there is a separate object for each float in the tuple.

We can also groups sequences by mutability:

  • Mutable sequences
    • list, bytearray, array.array, and collections.queue
  • Immutable sequences
    • tuple, str, bytes

Recall, the big difference is that mutable objects can have their values change after declaration. i.e. you can change the value of an element of a list but can do that in a tuple:

Code 
a = [1,2,3]
a[0] = 4
from numpy import remainder
print(a)

b = (1,2,3)
try:
  b[0] = 4
except Exception as e:
  print(e)
print(b)
[4, 2, 3]
'tuple' object does not support item assignment
(1, 2, 3)

List Comprehension and Generator Expressions

It’s the classic:

Code 
symbols = '$c$ao'

codes = []

for symbol in symbols:
  codes.append(ord(symbol))

# equivalent:

codes = [ord(symbol) for symbol in symbols]
codes
[36, 99, 36, 97, 111]

Coding tip: line breaks are ignored when using [], (), or {}. So you don’t need to use \ for multiline (which actually breaks if there is a space after the \). Also, a trailing comma is ignored when creating a series of comma separate values, so it is best practice to include it even at the end.

List comprehension, generator expressions, set and dict comprehension has a local scope to hold variables assigned in the for clause. However, variables assigned with the “Walrus Operator” remain accessible after those comprehension or expressions return - unlike local variables in a function. PEP 572 defines the scope of the target of := as the enclosing function, unless there is a global or nonlocal declaration for that target.

Code 
x = 'ABC'
codes = [ord(x) for x in x]
print(x)

codes = [last := ord(c) for c in x]
print(codes)
print(last)
try:
  print(c)
except Exception as e:
  print(e)
ABC
[65, 66, 67]
67
name 'c' is not defined

A little more about the Walrus Operator: it both assigns and returns the variable assigned. This is similar to how you can do if (cnt <- cnt + 1) in R and get the cnt checked in the if statement, because R will always sliently return the object of an assigment.

Code 
chunk = file.read(1024)
while chunk:
    process(chunk)
    chunk = file.read(1024)
    
while (chunk := file.read(1024)):
    process(chunk)

A simpler one:

Code 
try:
  print(a = 2)
except Exception as e:
  print(e)

print(a := 2)
'a' is an invalid keyword argument for print()
2

We can do similar comprehensions with filter and map, but readability suffers.

Listcomps versus map and filter

Listcomps do everything map and filter functions can do, without the challenging lambda functions:

Code 
asc = [ord(s) for s in symbols if ord(s) > 40]
print(asc)

asc = list(
  filter(
    lambda c: c > 40,
    map(ord, symbols)
  )
)
asc
[99, 97, 111]
[99, 97, 111]

Cartesian Products

Code 
colors = ['black', 'white']
sizes = ['S', 'M', 'L']
tshirts = [(color, size) for color in colors
                         for size in sizes]
print(tshirts)
print(len(colors)*len(sizes))
print(len(tshirts))

## Same as
# for color in colors:
#   for size in sizes:
#     print((color, sizes))
[('black', 'S'), ('black', 'M'), ('black', 'L'), ('white', 'S'), ('white', 'M'), ('white', 'L')]
6
6

Listcomps are a “one-trick pony”. For generating data, it is best to use a genexp.

Generator Expressions

A genexp can save memory because it yields items one by one using the iterator protocol instead of building a whole list just to feed another constructor. Genexp use the same syntax as listcomps but use () instead of []. Another note is that genexp are lazy and only evaluate when called. Listcomps are eager and exectue right away. Genexp can also be useful if the sequence generated is potentially infinite.

Tuples are not just immutable Lists

Tuples can also be used a records with no field names. When doing this, it usually means that the order and the number of items is fixed.

For example:

Code 
lax_coords = (33.99, -118.408)
city, year, pop, chg, area = ('Tokyo', 2003, 32_450, 0.66, 8014) # an example of tuple _unpacking_
traveler_ids = [('USA', '31195855'), ('BRA', 'CE342568'), ('ESP', 'XDA205856')]

for passport in sorted(traveler_ids):
  print('%s/%s' % passport)

for country, _ in traveler_ids:
  print(country)
BRA/CE342568
ESP/XDA205856
USA/31195855
USA
BRA
ESP

Notice the % operator understands tuples and treats each item as a separate field.

Also, be careful with the use of _ as a dummy variable. For match/case statementes it is treated as a dummy variable.

Tuples as immutable lists

Two things we get with using a tuple

  • Clarity: we know the length will never change
  • Performance: A tuple will use less memory than a list of the same length. It futher allows python to do some operations behind the scenes.

Note that the references in a tuple cannot be deleted or replace. But if they are changed, the tuple will follow that value and sometimes given unexpected results:

Code 
a = (10, 'alpha', [1,2])
b = (10, 'alpha', [1,2])
print(a==b)

b[-1].append(3)
a==b
True
False

We’ll go into more detail on what “Hashable” means later, but for now, just know that it means that we cannot change the value. For instance:

Code 
def fixed(o):
  try:
    hash(o)
  except TypeError:
    return False
  return True

tf = (10, 'alpha', (1,2))
tm = (10, 'alpha', [1,2])

print(fixed(tf))
print(fixed(tm))
True
False

Notes:

  • If you call tuple(t) on tuple t, it will just return a reference to tuple t. If intsead you call list(l) on list l, it will have to make a copy of l.
  • tuple supports all list methods except for those that add or remove items
    • tuple also lacks a __reversed__ method, but that is just for optimization. You can still call reversed(t) on a tuple.
    • Full list is given on page 34

Unpacking sequences and iterables

Unpacking works on anything that is iterable, even iterators (although they dont support indexing).

Code 
lax_coords = (33.9425, -118.40805)
lat, lon = lax_coords # unpacking
lat
33.9425

You can also use unpacking to swap values of variables: b, a = a, b

using * for unpacking

Another example is using * when calling a function:

Code 
divmod(20, 8)
(2, 4)
Code 
t = (20, 8)
divmod(*t)
(2, 4)
Code 
quotient, remainder = divmod(*t)
a = quotient, remainder # pack it back up
print(type(a))
<class 'tuple'>

You’ve seen *args to grab excess assigments in python functions, but you can also use it to grab excess items in unpacking:

Code 
a, b, *rest = range(5)
rest # comes as a list
[2, 3, 4]
Code 
a, b, *rest = range(3)
a, b, *rest
(0, 1, 2)
Code 
try:
  a, b, rest = range(2)
except Exception as e:
  print(e)

a, b, *rest = range(2)
a, b, rest
not enough values to unpack (expected 3, got 2)
(0, 1, [])

Can also it at the start too:

Code 
*head, a, b = range(5)
head, a, b
([0, 1, 2], 3, 4)

You can also unpack function calls :

Code 
def fun(a,b,c, *rest):
  return a, b, c, rest

fun(*[1,2], 3, *range(4,7))
(1, 2, 3, (4, 5, 6))

You can also use it for defining lists, tuples, or set literals:

Code 
*range(4), 4
(0, 1, 2, 3, 4)
Code 
[*range(4), 4]
[0, 1, 2, 3, 4]

Nested unpacking

We can use nesting if the value has the same nesting structure, for instance:

Code 
a, b, c, (d, e) = (1, 2, 3, (4, 5))
d
4

One last tuple thing, don’t forget that when defining tuples with a single item, they must have a trailing comma. Otherwise it just unpacks and becomes that value.

Code 
a = (4)
type(a)
int
Code 
a = (4,)
type(a)
tuple

Pattern matching and sequences

Let’s say that we have some data coming in that we need to perform an action in response. To, if this data is mixed types, we might use a match method:

Code 
def handle(self, message):
  match message:
    case ['BEEPER', frequency, times]:
      self.beep(times, frequency)
    case ['NECK', angle]:
      self.rotate_neck(angle)
    case ['LED', ident, intensity]:
      self.leds[ident].set_brightness(ident, intensity)
    case ['LED', ident, red, green, blue]:
      self.leds[ident].set_color(ident, red, green, blue)
    case _: # wildcard and default case
      raise InvalidCommand(message)
  • The expression after match is the subject which python will try to match in the case clause.
  • The first patter matches anything that is a sequence of three items, where the first item is the string 'BEEPER'
    • The second and third item can be anything and will be bound to the variables frequency and times
    • Also like the first entry here is the type of message and the other items are the metadata
  • Similarly, the second pattern must start with 'NECK' and contain 2 total items (including NECK)

This looks very similar to the C-style switch/case, but this Python style also supports destructuring - a more advanced form of unpacking.

Code 
metro_areas = [
  ('Tokyo', 'JP', 36.933, (35.689722, 139.691667)),
  ('Delhi NCR', 'IN', 21.935, (28.613889, 77.208889)),
  ('Mexico City', 'MX', 20.142, (19.433333, -99.133333)),
  ('New York-Newark', 'US', 20.104, (40.808611, -74.020386)),
  ('São Paulo', 'BR', 19.649, (-23.547778, -46.635833)),
]

def main():
  print(f'{"":15} | {"latitude":>9} | {"longitude":>9}')
  for record in metro_areas:
    match record:
      case [name, _, _, (lat, lon)] if lon <= 0:
        print(f'{name:15} | {lat:9.4f} | {lon:9.4f}')

main()

# Notice we can use :15 if the f-string to set the width of the space
                |  latitude | longitude
Mexico City     |   19.4333 |  -99.1333
New York-Newark |   40.8086 |  -74.0204
São Paulo       |  -23.5478 |  -46.6358

Let’s unpack (lol) above:

  • The subject of the match is record
  • case statement has the pattern - in this call all records should match
    • It also has an optional guard with the if keyword

in general, a pattern is matched if:

  • The subject is a sequence
  • The subject and the pattern have the same number of sequences
  • Each corresponding item matches, including nested items

For the example pattern [name, _, _, (lat, lon)], there are 4 items and the last item must be a two-item sequence.

Note you can use lists or tuples - to the sequence matching they mean the same thing.
